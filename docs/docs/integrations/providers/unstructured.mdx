# Unstructured | 非構造化

> [Unstructured.IO](https://www.unstructured.io/)の`unstructured`パッケージは、PDFやWordドキュメントのような生のソースドキュメントからクリーンなテキストを抽出します。このページでは、LangChain内で[`unstructured`](https://github.com/Unstructured-IO/unstructured)エコシステムをどのように使用するかについて説明しています。
>
> > The `unstructured` package from
> > [Unstructured.IO](https://www.unstructured.io/) extracts clean text from raw source documents like
> > PDFs and Word documents.
> > This page covers how to use the [`unstructured`](https://github.com/Unstructured-IO/unstructured)
> > ecosystem within LangChain.

## Installation and Setup | インストールとセットアップ

ローカルで実行されるローダーを使用している場合は、`unstructured`とその依存関係をローカルで実行するための以下の手順に従ってください。

> If you are using a loader that runs locally, use the following steps to get `unstructured` and
> its dependencies running locally.

* `pip install unstructured`を使用してPython SDKをインストールしてください。
  > Install the Python SDK with `pip install unstructured`.
  * 特定のドキュメントに関連する依存関係はextrasを使ってインストールできます。例えば、`pip install "unstructured[docx]"`のようにします。
    > You can install document specific dependencies with extras, i.e. `pip install "unstructured[docx]"`.
  * すべてのドキュメントタイプの依存関係をインストールするには、`pip install "unstructured[all-docs]"`を使用してください。
    > To install the dependencies for all document types, use `pip install "unstructured[all-docs]"`.
* 以下のシステム依存関係を、まだシステムにインストールされていない場合はインストールしてください。解析する文書のタイプによっては、これらすべてが必要でない場合もあります。
  > Install the following system dependencies if they are not already available on your system.
  > Depending on what document types you're parsing, you may not need all of these.
  * `libmagic-dev`（ファイルタイプ検出）
    > `libmagic-dev` (filetype detection)
  * `poppler-utils`（画像とPDF用）
    > `poppler-utils` (images and PDFs)
  * `tesseract-ocr`（画像やPDF）
    > `tesseract-ocr`(images and PDFs)
  * `libreoffice`（MS Officeのドキュメント）
    > `libreoffice` (MS Office docs)
  * `pandoc`（EPUBs）
    > `pandoc` (EPUBs)

設定を最小限に抑えてすぐに始めたい場合は、`pip install unstructured`を実行して`UnstructuredAPIFileLoader`または`UnstructuredAPIFileIOLoader`を使用することができます。これにより、ホストされたUnstructured APIを使用してドキュメントを処理できます。

> If you want to get up and running with less set up, you can
> simply run `pip install unstructured` and use `UnstructuredAPIFileLoader` or
> `UnstructuredAPIFileIOLoader`. That will process your document using the hosted Unstructured API.

Unstructured APIを使用するにはAPIキーが必要です。無料のAPIキーは[こちら](https://www.unstructured.io/api-key)から生成でき、今すぐ使用を開始できます！APIコールを始めるためには、[こちら](https://github.com/Unstructured-IO/unstructured-api)のREADMEをチェックしてください。フィードバックをお待ちしておりますので、[コミュニティのSlack](https://join.slack.com/t/unstructuredw-kbe4326/shared_invite/zt-1x7cgo0pg-PTptXWylzPQF9xZolzCnwQ)でどのように進んでいるかお知らせください。品質とパフォーマンスの向上にご期待ください！Unstructured APIを自己ホストするかローカルで実行したい場合は、[こちら](https://github.com/Unstructured-IO/unstructured-api#dizzy-instructions-for-using-the-docker-image)の指示を確認してください。

## Wrappers | ラッパー

### Data Loaders | データローダー

`langchain`内の主要な`unstructured`ラッパーはデータローダーです。以下に、最も基本的な非構造化データローダーの使用方法を示します。`langchain.document_loaders`モジュールには、その他にもファイル固有のデータローダーが用意されています。

> The primary `unstructured` wrappers within `langchain` are data loaders. The following
> shows how to use the most basic unstructured data loader. There are other file-specific
> data loaders available in the `langchain.document_loaders` module.

```python
from langchain.document_loaders import UnstructuredFileLoader

loader = UnstructuredFileLoader("state_of_the_union.txt")
loader.load()
```

`UnstructuredFileLoader(mode="elements")`でローダーをインスタンス化すると、情報が利用可能であれば、ページ番号やテキストタイプ（例えば、タイトル、ナラティブテキスト）などの追加メタデータを追跡するようになります。

> If you instantiate the loader with `UnstructuredFileLoader(mode="elements")`, the loader
> will track additional metadata like the page number and text type (i.e. title, narrative text)
> when that information is available.
