# Welcome Contributors | 貢献者の皆さん、ようこそ

こんにちは！LangChainへの貢献に興味を持っていただき、心から感謝しています。私たちは急速に進化する分野でのオープンソースプロジェクトであり、新機能の追加、インフラの改善、ドキュメントの充実、バグ修正など、あらゆる貢献を大いに歓迎しています。

> Hi there! Thank you for even being interested in contributing to LangChain.
> As an open-source project in a rapidly developing field, we are extremely open to contributions, whether they involve new features, improved infrastructure, better documentation, or bug fixes.

## 🗺️ Guidelines | 🗺️ ガイドライン

### 👩‍💻 Ways to contribute | 👩‍💻 貢献する方法

LangChainへの貢献方法は様々です。ここに、人々がよく行う貢献の方法をいくつか紹介します：

> There are many ways to contribute to LangChain. Here are some common ways people contribute:

* [**ドキュメンテーション**](./documentation): このドキュメントを含む、私たちのドキュメントの改善にご協力をお願いします！
  > [**Documentation**](./documentation): Help improve our docs, including this one!
* [**コード**](./code): コードの作成、バグの修正、またはインフラストラクチャの改善を手伝ってください。
  > [**Code**](./code): Help us write code, fix bugs, or improve our infrastructure.
* [**統合**](./integrations): お気に入りのベンダーやツールとの統合にご協力ください。
  > [**Integrations**](./integrations): Help us integrate with your favorite vendors and tools.

### 🚩GitHub Issues | 🚩GitHubのイシュー

私たちの[イシュー](https://github.com/langchain-ai/langchain/issues)ページは、バグ、改善提案、機能リクエストを常に最新の情報に更新しています。

> Our [issues](https://github.com/langchain-ai/langchain/issues) page is kept up to date with bugs, improvements, and feature requests.

関心のある問題を整理し、発見するために役立つラベルの分類体系があります。これらを利用して問題を整理する際にご活用ください。

> There is a taxonomy of labels to help with sorting and discovery of issues of interest. Please use these to help organize issues.

問題に取り組み始めたら、自分をその問題の担当者に設定してください。

> If you start working on an issue, please assign it to yourself.

問題を追加する際には、単一のモジュラーなバグ、改善点、または機能に焦点を当ててください。2つの問題が関連している、または相互にブロックしている場合は、それらを結合するのではなく、リンクしてください。

> If you are adding an issue, please try to keep it focused on a single, modular bug/improvement/feature.
> If two issues are related, or blocking, please link them rather than combining them.

私たちは、これらの問題を可能な限り最新の状態に保つよう努めますが、この分野が急速に発展しているため、いくつかの問題が時代遅れになる可能性があります。そのようなことに気づいた場合は、どうぞお知らせください。

> We will try to keep these issues as up-to-date as possible, though
> with the rapid rate of development in this field some may get out of date.
> If you notice this happening, please let us know.

### 🙋Getting Help | 🙋サポートを得る

私たちの目指すところは、可能な限りシンプルな開発者セットアップを提供することです。セットアップに何か問題が生じた場合は、メンテナーにご連絡ください。私たちはあなたが行き詰まりなく作業を進められるよう支援するだけではなく、将来の貢献者がスムーズに作業できるようなプロセスを確立することも重視しています。

> Our goal is to have the simplest developer setup possible. Should you experience any difficulty getting setup, please
> contact a maintainer! Not only do we want to help get you unblocked, but we also want to make sure that the process is
> smooth for future contributors.

同様に、私たちはコードベースにおいて一定のリンティング、フォーマッティング、ドキュメンテーションの基準を守ることを徹底しています。これらの基準に従うことが困難だと感じたり、単に面倒だと感じる場合は、遠慮なくメンテナーに連絡して支援を求めてください。私たちは、これらの基準が優れたコードをコードベースに組み込むことの障害となることを望んでいません。

> In a similar vein, we do enforce certain linting, formatting, and documentation standards in the codebase.
> If you are finding these difficult (or even just annoying) to work with, feel free to contact a maintainer for help -
> we do not want these to get in the way of getting good code into the codebase.
