## Chat models | チャットモデル

### Google AI | Google AI

`ChatGoogleGenerativeAI` クラスを通じて、`gemini-pro` や `gemini-pro-vision` などの GoogleAI `Gemini` モデルにアクセスできます。

> Access GoogleAI `Gemini` models such as `gemini-pro` and `gemini-pro-vision` through the `ChatGoogleGenerativeAI` class.

```bash
pip install -U langchain-google-genai
```

APIキーを設定してください。

> Configure your API key.

```bash
export GOOGLE_API_KEY=your-api-key
```

```python
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(model="gemini-pro")
llm.invoke("Sing a ballad of LangChain.")
```

Geminiビジョンモデルは、単一のチャットメッセージを提供する際に画像入力をサポートしています。例：

> Gemini vision model supports image inputs when providing a single chat message. Example:

```python
from langchain_core.messages import HumanMessage
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(model="gemini-pro-vision")
# example
message = HumanMessage(
    content=[
        {
            "type": "text",
            "text": "What's in this image?",
        },  # You can optionally provide text parts
        {"type": "image_url", "image_url": "https://picsum.photos/seed/picsum/200/300"},
    ]
)
llm.invoke([message])
```

image\_urlの値は、以下のいずれかにすることができます：

> The value of image\_url can be any of the following:

* 公開画像のURL
  > A public image URL
* GCSファイル（例："gcs://path/to/file.png"）
  > A gcs file (e.g., "gcs://path/to/file.png")
* ローカルファイルパス
  > A local file path
* Base64でエンコードされた画像（例：data:image/png;base64,abcd124）
  > A base64 encoded image (e.g., data:image/png;base64,abcd124)
* PILイメージ
  > A PIL image

### Google ScaNN | Google ScaNN

> > [Google ScaNN](https://github.com/google-research/google-research/tree/master/scann)

>
> `ScaNN`は、大規模なベクトル類似性検索を効率的に行うための方法です。
>
> > `ScaNN` is a method for efficient vector similarity search at scale.

> `ScaNN`は、最大内積検索のための検索空間の剪定と量子化を含んでおり、ユークリッド距離などの他の距離関数にも対応しています。実装はAVX2をサポートするx86プロセッサ向けに最適化されています。詳細については、[Google Research github](https://github.com/google-research/google-research/tree/master/scann)をご覧ください。

> > `ScaNN` includes search space pruning and quantization for Maximum Inner
> > Product Search and also supports other distance functions such as

> > for more details.

## Document Transformers | ドキュメントトランスフォーマー

### Google Document AI | Google Document AI

> [Document AI](https://cloud.google.com/document-ai/docs/overview)は、文書の非構造化データを構造化データに変換する`Google Cloud Platform`サービスであり、それによってデータの理解、分析、そして活用が容易になります。
>
> > [Document AI](https://cloud.google.com/document-ai/docs/overview) is a `Google Cloud Platform`
> > service that transforms unstructured data from documents into structured data, making it easier
> > to understand, analyze, and consume.

Google Cloud Storage（GCS）バケットを設定し、独自のOCRプロセッサを[作成する必要があります](https://cloud.google.com/document-ai/docs/create-processor)。`GCS_OUTPUT_PATH`はGCS上のフォルダへのパス（`gs://`で始まる）である必要があり、プロセッサ名は`projects/PROJECT_NUMBER/locations/LOCATION/processors/PROCESSOR_ID`の形式であるべきです。これはプログラムによって取得することも、Google Cloud Consoleの`Processor details`タブにある`Prediction endpoint`セクションからコピーすることも可能です。

> We need to set up a [`GCS` bucket and create your own OCR processor](https://cloud.google.com/document-ai/docs/create-processor)\
> The `GCS_OUTPUT_PATH` should be a path to a folder on GCS (starting with `gs://`)
> and a processor name should look like `projects/PROJECT_NUMBER/locations/LOCATION/processors/PROCESSOR_ID`.
> We can get it either programmatically or copy from the `Prediction endpoint` section of the `Processor details`
> tab in the Google Cloud Console.

```bash
pip install google-cloud-documentai
pip install google-cloud-documentai-toolbox
```

[使用例](/docs/integrations/document_transformers/docai)をご覧ください。
