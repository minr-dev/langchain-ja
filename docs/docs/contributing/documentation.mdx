# Contribute Documentation | ドキュメンテーションに貢献する

docsディレクトリには、ドキュメンテーションとAPIリファレンスが含まれています。

> The docs directory contains Documentation and API Reference.

ドキュメンテーションは、[Quarto](https://quarto.org)と[Docusaurus 2](https://docusaurus.io/)を用いて構築されます。

> Documentation is built using [Quarto](https://quarto.org) and [Docusaurus 2](https://docusaurus.io/).

APIリファレンスは、主にコードから[sphinx](https://www.sphinx-doc.org/en/master/)によって自動生成され、[Read the Docs](https://readthedocs.org/)で公開されています。そのため、全てのクラスとメソッドに適切なドキュメントを追加していただくようお願い申し上げます。

> API Reference are largely autogenerated by [sphinx](https://www.sphinx-doc.org/en/master/) from the code and are hosted by [Read the Docs](https://readthedocs.org/).
> For that reason, we ask that you add good documentation to all classes and methods.

コード整形と同じように、ドキュメンテーションが面倒であることを我々は認識しています。もしそれを行いたくなければ、プロジェクトのメンテナーにご連絡ください。彼らがあなたをサポートします。私たちは、優れたコードの貢献が障害になることを望んでいません。

> Similar to linting, we recognize documentation can be annoying. If you do not want to do it, please contact a project maintainer, and they can help you with it. We do not want this to be a blocker for good code getting contributed.

## Build Documentation Locally | ローカル環境でドキュメントをビルドする

### Install dependencies | 依存関係をインストールする

* [Quarto](https://quarto.org) - Jupyterノートブック（`.ipynb`ファイル）をDocusaurusで公開する際にmdxファイルへ変換するパッケージです。
  > [Quarto](https://quarto.org) - package that converts Jupyter notebooks (`.ipynb` files) into mdx files for serving in Docusaurus.
* モノレポジトリのルートで`poetry install`を実行してください。
  > `poetry install` from the monorepo root

### Building | ドキュメント構築

以下のコマンドにおいて、接頭辞 `api_` はそれがAPIリファレンス用の操作であることを示しています。

> In the following commands, the prefix `api_` indicates that those are operations for the API Reference.

ドキュメントのビルドを始める前に、ビルドディレクトリをクリーンアップすることが常に良いとされています：

> Before building the documentation, it is always a good idea to clean the build directory:

```bash
make docs_clean
make api_docs_clean
```

次に、下記の手順に従ってドキュメントを構築することができます：

> Next, you can build the documentation as outlined below:

```bash
make docs_build
make api_docs_build
```

最後に、すべてのリンクが有効であることを確認するためにリンクチェッカーを実行してください：

> Finally, run the link checker to ensure all links are valid:

```bash
make docs_linkcheck
make api_docs_linkcheck
```

### Linting and Formatting | リントとフォーマッティング

ドキュメントはモノレポジトリのルートからリント処理されます。ドキュメントをリントするには、そのディレクトリで以下のコマンドを実行してください：

> The docs are linted from the monorepo root. To lint the docs, run the following from there:

```bash
poetry install --with lint,typing
make lint
```

書式に関するエラーがある場合、以下のコマンドで自動的に修正できます：

> If you have formatting-related errors, you can fix them automatically with:

```bash
make format
```

## Verify Documentation changes | ドキュメンテーションの変更を確認する

リポジトリにドキュメントの変更をプッシュした後、プルリクエストの「Conversation」ページにある「View deployment」または「Visit Preview」ボタンをクリックして、変更が意図したものであるかをプレビューで確認し、検証することができます。この操作により、ドキュメントの変更内容をプレビューすることができます。このプレビューは[Vercel](https://vercel.com/docs/getting-started-with-vercel)によって生成されます。

> After pushing documentation changes to the repository, you can preview and verify that the changes are
> what you wanted by clicking the `View deployment` or `Visit Preview` buttons on the pull request `Conversation` page.
> This will take you to a preview of the documentation changes.
> This preview is created by [Vercel](https://vercel.com/docs/getting-started-with-vercel).
