# 📕 Package Versioning | 📕 パッケージバージョニング

現在、LangChainは特定の事情に応じたリリースプロセスを採用しており、メンテナーが頻繁にリリースを行い、[PyPI](https://pypi.org/)に公開しています。各パッケージは少しずつ異なるバージョニングが施されています。

> As of now, LangChain has an ad hoc release process: releases are cut with high frequency by
> a maintainer and published to [PyPI](https://pypi.org/).
> The different packages are versioned slightly differently.

## `langchain-core` | `langchain-core`

`langchain-core`は現在、バージョン`0.1.x`を採用しています。

> `langchain-core` is currently on version `0.1.x`.

`langchain-core`はLangChainエコシステム全体の基本的な抽象概念とランタイムを含んでいるため、重大な変更がある場合は事前に通知し、バージョンを更新します。ただし、`langchain_core.beta`に含まれるものはこの限りではありません。`langchain_core.beta`が存在する理由は、この分野の変化の速度を考慮して、迅速に進めることが依然として優先されているためであり、このモジュールはその迅速な対応を可能にするための試みです。

> As `langchain-core` contains the base abstractions and runtime for the whole LangChain ecosystem, we will communicate any breaking changes with advance notice and version bumps. The exception for this is anything in `langchain_core.beta`. The reason for `langchain_core.beta` is that given the rate of change of the field, being able to move quickly is still a priority, and this module is our attempt to do so.

マイナーバージョンの増加は以下の場合に行われます：

> Minor version increases will occur for:

* `langchain_core.beta`には含まれていない公開インターフェースに対する破壊的変更
  > Breaking changes for any public interfaces NOT in `langchain_core.beta`

パッチバージョンの増加は以下の場合に行われます：

> Patch version increases will occur for:

* バグ修正
  > Bug fixes
* 新機能
  > New features
* プライベートインターフェースの変更
  > Any changes to private interfaces
* `langchain_core.beta`に対する変更
  > Any changes to `langchain_core.beta`

## `langchain` | 📕 パッケージバージョニング

`langchain`は現在、バージョン`0.0.x`になっています。

> `langchain` is currently on version `0.0.x`

全ての変更はパッチバージョンの増加を伴います。公開インターフェースに対する変更は、ほとんど常に後方互換性を保つ形で実施されますが、後方互換性が保たれない場合は、変更が行われる前に事前に通知されます。

> All changes will be accompanied by a patch version increase. Any changes to public interfaces are nearly always done in a backwards compatible way and will be communicated ahead of time when they are not backwards compatible.

私たちは、2024年1月に`langchain` v0.1をリリースすることを目標としており、その時点で`langchain`は`langchain-core`と同様のバージョニングポリシーを採用する予定です。

> We are targeting January 2024 for a release of `langchain` v0.1, at which point `langchain` will adopt the same versioning policy as `langchain-core`.
